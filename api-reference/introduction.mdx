## Authentication

All API endpoints are authenticated using an API key sent in the `X-API-Key` header and picked up from the specification file.

```json
"security": [
  {
    "apiKeyAuth": []
  }
]
```

## Job workflow

Identity research runs asynchronously:

1. Send a `POST` request to `/research-identity` with `name` and `website` to start a job. The response includes a `job_id`.
2. Poll `GET /jobs/{job_id}` until `status` is `completed`.
3. When complete, read `result.kernel_id` and `result.legal_name` from the response.

While the job is in progress, the status will be `pending` or `processing`. If the job fails, `status` will be `failed`.
